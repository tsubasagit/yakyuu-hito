## 今日の打ち合わせ要約

### 商談サマリ
| 項目 | 内容 |
|---|---|
| 発注元 | 株式会社ひと（LINE Yahoo / データスタジアム 共同の大学野球配信事業） |
| 対象 | 大学生スタッフが撮影・配信を兼任する現場 |
| 予算 | **¥200,000 買い切り** |
| 納期 | **2026-08末** 完納（2026-07末 開発完了見込み） |
| 本番投入 | **2026-09 秋季リーグ** |
| 次の宿題 | **5/1 までに 20万円内の最終提案** を送付 |
| コミュニケーション | Facebook Messenger（坂井氏がグループ作成） |

### なぜ今回の案件になったか
- 原田氏が既存 `yakyuu` v0.3.0 を OBS で試用 →「使えそう」と判断
- 市販ツールは「高機能で学生に使いこなせない」または「安価で品質低」
- JVCカメラ内蔵スコア入力は無線不安定・カメラ近傍拘束の制約

### 合意した方向性
「学生がワンクリックで操作できる、必要十分の4要素（BSO・スコアボード・打者名・スタメン）を独立トグル＋自由配置できるOBSオーバーレイ」

## Phase 1 実装スコープ（¥200,000 / 2026-08末）

### ✅ 必須4要素（表示単位）

| # | 要素 | 表示内容 | 個別ON/OFF |
|---|---|---|---|
| 1 | **BSO** | ボール・ストライク・アウト ランプ | 必須 |
| 2 | **スコアボード** | イニング別得点（1〜12回＋R） | 必須 |
| 3 | **打者名** | チーム略称＋選手名＋学年 | 必須 |
| 4 | **スタメン一覧** | 9名＋ポジション＋学年 | 必須 |

※ モックアップ（`docs/design/overlay-mockup.jpg`）準拠で以下3要素も同時提供（付加価値）:
- ミニスコア / 大型スコア / 大会タイトル

### ✅ 必須UI機能

- [ ] **個別表示トグル** — コントロール画面上段に並ぶ大きなボタンで1クリックON/OFF
- [ ] **X/Y自由配置** — ドラッグ配置、localStorage 永続化、リセット機能
- [ ] **選手登録CSVインポート** — サンプルレイアウト付き（`docs/sample-roster.csv` 提供）
- [ ] **チームカラー HEX入力** — 学生が色をメモして再利用可
- [ ] **代打発表カード** — 選手名＋学年＋一行コメント（「少年クラブ優勝経験あり」等）
- [ ] **大会情報入力** — 大会名／副題／対戦／会場／日付

### ✅ 動作要件

- [x] **完全オフライン動作**（初回ロード以降は電波不要）
- [ ] **OBSブラウザソース対応**（透明背景、1920×1080）
- [ ] **Aチーム背景色固定** 等の個別調整対応
- [ ] **学生運用UX**（48×48px以上の大型ボタン、日本語ラベル、誤操作防止）

### ✅ エフェクト（最小仕様）

- [ ] ホームラン等のフェードイン／フェードアウト
- [ ] 指定フォルダからの画像呼び出し
- [ ] 連盟未使用時のために表示ON/OFF切替必須

### ❌ Phase 1 で**削除・非搭載**

yakyuu v0.3.0 から持ち込まない機能:
- マスコット表示
- プレーバイプレー経過ログ
- 経過時間タイマー
- 打率・HR・打点・OPS 等の詳細打撃スタッツ
- 登板数・勝敗・投球数 等の詳細投手スタッツ
- 試合前待機画面（「まもなく試合開始」）
- 速報テロップ（ティッカー）
- 10種エフェクトアニメーション（上記の最小仕様のみ採用）
