# yakyuu-hito - サービス仕様書

## 1. サービス概要
- **一言で**: 大学野球オンライン配信向け、OBSブラウザソース用スコアボード／テロップオーバーレイ（株式会社ひと 専用カスタマイズ版）
- **対象ユーザー**: 株式会社ひとが大学野球の配信運用を委託する大学生スタッフ（スコアキーパー兼配信オペレーター）
- **解決する課題**:
  - JVCカメラ内蔵スコア入力アプリは無線接続が不安定・操作者がカメラ近傍に拘束される
  - 市販ツールは高機能すぎて学生に使いこなせない、または安価版は品質が低い
  - 既存 `yakyuu` (v0.3.0) は機能が多く、学生運用向けに不要機能が混在している
- **ステータス**: v0.4.0-alpha（2026-04-23 キックオフ、2026-08 完納目標、2026-09 秋季リーグ本番投入）
- **派生元**: [tsubasagit/yakyuu](https://github.com/tsubasagit/yakyuu) v0.3.0 を株式会社ひと向けに分岐カスタマイズ

## 2. ユーザーロールと権限
| ロール | できること |
|---|---|
| 配信オペレーター（学生スタッフ） | `/control` で試合データ入力、各オーバーレイ要素のON/OFF、CSVで選手データインポート |
| 視聴者 | LINE系スポーツナビ等の配信経由で `/overlay` の映像オーバーレイを視聴 |
| 発注者（株式会社ひと） | 仕様変更ディレクション、運用フィードバック |

## 3. 機能一覧

### 必須4要素（MTG 2026-04-23 合意）
- [ ] **BSO（ボール・ストライク・アウト）単独表示** — スコアボードと独立してON/OFF可能
- [ ] **スコアボード単独表示** — イニング進行・得点、イニング間／グランド整備中のみ表示する運用に対応
- [ ] **打者名単独表示** — 打率等の詳細は非表示。「誰が打席に立っているか」のみ
- [ ] **スターティングラインナップ一覧** — 打順表示、代打は打者名表示切替で対応

### 必須UI要件
- [ ] 要素ごとの表示/非表示トグルボタン（`/control` 上で1クリック切替）
- [ ] 各要素のX/Y座標自由配置（ドラッグ or 数値入力）
- [ ] 選手登録CSVインポート（サンプルレイアウト提供要）
- [ ] チームカラーのカラーコード（HEX）入力対応
- [ ] オフライン動作（初回ロード時のみネット必要）

### 削除・非表示対象（学生運用のため簡素化）
- [ ] マスコット表示機能
- [ ] 経過ログ（プレーバイプレー）
- [ ] 経過時間タイマー
- [ ] 打率・HR・打点・OPS等の打撃詳細スタッツ
- [ ] 登板数・勝敗・投球数等の投手詳細スタッツ（最小限に）

### エフェクト（最小仕様）
- [ ] ホームラン等のフェードイン／フェードアウトエフェクト
- [ ] 指定フォルダからの画像呼び出し表示
- [ ] 連盟によっては未使用のため、必ず非表示切替可能

### 継承機能（yakyuu v0.3.0から引き継ぎ）
- [x] React + Vite + TypeScript + Tailwind CSS + Zustand
- [x] BroadcastChannel API によるタブ間リアルタイム同期
- [x] localStorage による状態永続化（オフライン動作）
- [x] ドラッグ可能パネル
- [x] デモデータプリセット
- [x] CSVインポート機能（要レイアウト見直し）

## 4. 画面一覧
| 画面 | パス | 対象ロール | 概要 |
|---|---|---|---|
| ホーム | `/` | 配信オペレーター | プロジェクト説明・リンク |
| コントロールパネル | `/#/control` | 配信オペレーター | 試合データ入力、各要素トグル、CSVインポート、カラーコード設定 |
| オーバーレイ | `/#/overlay` | 視聴者（OBS経由） | 透明背景・4要素を重畳表示 |

## 5. データモデル
### 主要ステート（Zustand store）
- `GameState` — チーム／イニング／BSOカウント／走者／選手情報
- `UIVisibility` — 要素別の表示フラグ（bso, scoreboard, batter, lineup, etc.）
- `PanelLayout` — 各要素のX/Y座標・サイズ
- `TeamStyle` — チーム名・略称・カラーコード（HEX）

### CSV選手登録レイアウト（暫定案・サンプル提供予定）
```csv
team_id,order,position,uniform_no,player_name
home,1,中,1,佐藤太郎
home,2,右,5,鈴木一郎
...
```

## 6. 外部連携
- **ホスティング**: GitHub Pages（`tsubasagit.github.io/yakyuu-hito/` 想定）
- **認証**: なし（ローカルストレージのみ）
- **DB**: なし（サーバーレス）
- **OBS連携**: ブラウザソースとして `/#/overlay` を1920x1080で読み込み

## 7. ビジネスルール
- **固定予算**: 20万円（買い切り、MTG 2026-04-23 合意）
- **納期**: 2026-08末（2026-09 秋季リーグ投入）。開発完了は2026-07末目処
- **知財**: yakyuu OSS（MIT）からの派生。株式会社ひと 向けカスタマイズ部分の扱いは別途協議
- **コミュニケーション**: Facebook Messenger（メールではなく Messenger を主体）

## 8. 非機能要件
- **想定ユーザー数**: 大学野球配信 1試合あたりオペレーター1〜2名。リーグ戦会期中は週数試合同時並行の可能性
- **パフォーマンス**: OBS 1920x1080 ブラウザソースで60fps表示を阻害しないこと
- **オフライン**: 初回ロード以降は完全オフライン動作（球場は電波不安定）
- **セキュリティ**: 公開情報のみ扱うため最小限。学生間の誤操作防止UIを重視

## 9. 既知の課題・制限
- yakyuu v0.3.0 から機能削減するため、既存コードの clean-up 工数が必要
- BroadcastChannel API は同一ブラウザ内タブでのみ動作。OBSブラウザソースとの同期は WindowCapture 方式を推奨
- デザイン最終形（WBC風）は宮崎側にセンス不安あり、画像参照・カラーコード運用で調整
